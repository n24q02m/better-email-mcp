💡 **What:** Changed `archiveFolderCache` to store Promises (`Map<string, string | Promise<string>>`) during the async `listFolders` lookup phase, converting the naive string cache into a concurrency-safe cache.
🎯 **Why:** To prevent concurrent archive actions (e.g., processing multiple emails from an LLM tool call array) from repeatedly hitting the IMAP server with `listFolders` calls while the first network call is still pending. This avoids unnecessary I/O roundtrips and cache stampedes.
📊 **Measured Improvement:** In local benchmarks, 5 concurrent requests previously resulted in 5 identical `listFolders` I/O calls and took ~154ms. After the fix, 5 concurrent requests hit the network exactly 1 time while yielding the same execution time, directly avoiding 4 redundant IMAP roundtrips!
