## 2025-02-12 - Inline Validation Focus for Custom Validation Forms
**Learning:** When using `novalidate` on HTML forms and implementing custom `invalid` event listeners to display inline errors (preventing the native browser error bubbles), standard form submission (`submit` event) does not automatically trigger those `invalid` events or focus invalid fields. The `submit` event simply fires regardless of validity, and if `event.preventDefault()` is used immediately to handle logic manually, the user receives no feedback if required fields are missing.
**Action:** Explicitly call `form.checkValidity()` in the custom `submit` handler to fire the `invalid` events and populate inline error messages. Additionally, manually focus the first invalid element using `form.querySelector('input:invalid, select:invalid, textarea:invalid')?.focus()` to ensure accessibility for keyboard and screen-reader users, as native focus behavior is lost when the default validation UI is bypassed.
