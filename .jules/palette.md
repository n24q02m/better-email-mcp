## 2024-05-30 - [Manual Validation Focus Trap]
**Learning:** When using `form.checkValidity()` to intercept form submissions and programmatically focus invalid elements, using a generic `form.querySelector(":invalid")` selector can incorrectly return non-focusable container elements like `<fieldset>` or `<form>`, causing the focus shift to fail silently.
**Action:** Always target specific focusable inputs when managing validation focus manually (e.g., `form.querySelectorAll("input:invalid, select:invalid, textarea:invalid")[0]`) to ensure keyboard/screen reader users are correctly guided to the error.
