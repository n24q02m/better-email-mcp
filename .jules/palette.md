## 2025-03-09 - Improving Accessible Feedback and Structure in Dynamic Forms
**Learning:** When generating UI elements dynamically without external UI libraries or component frameworks, typical CSS resets (`* { margin: 0; padding: 0; box-sizing: border-box; }`) easily obscure the semantic purpose of heading elements. Using generic tags (`<span>`, `<p>`) for titles rather than clear hierarchical headings (`<h2>`, `<h3>`) deprives screen reader users of structural landmarks they use to navigate a form's sections, and native `disabled` buttons alone fail to convey the "processing" state without an accompanying `aria-busy` attribute.
**Action:** Always map logical sections (like "Account 1") to their corresponding semantic heading tags (`<h3>`), rather than styling `<span>` tags to look like headings. For async buttons, toggle `aria-busy="true"` alongside `disabled="true"` to explicitly signal to screen readers that an operation is actively processing, and ensure custom buttons have `:focus-visible` styles so keyboard users do not lose their place.
