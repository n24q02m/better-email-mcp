## 2026-03-15 - Optimize HTML snippet extraction text processing
 **Learning:** In high-frequency text processing operations, avoiding repeated \`.toLowerCase()\` method calls for entity lookup by pre-computing or checking the raw case first can significantly reduce execution time and garbage collection overhead. Furthermore, combining multiple string/regex replacements into a single combined regex using backreferences (e.g., combining \`<script>\` and \`<style>\` tag stripping) dramatically improves the parsing overhead inside hot loops.
 **Action:** Updated \`html-utils.ts\` to use a combined \`/<(style|script)\\b[^>]*>[\\s\\S]*?(?:<\\/\\1\\b[^>]*>|$)/gi\` regex for faster stripping, and optimized \`ENTITY_MAP\` lookups by checking raw case before allocating lowercase strings.
