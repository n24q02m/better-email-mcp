🎯 **What:** The untested `textToHtml` utility function in `src/tools/helpers/smtp-client.ts` was missing dedicated tests, which is a gap given that it processes user input (Markdown into sanitized HTML) for outgoing emails.

📊 **Coverage:** A new test suite `src/tools/helpers/smtp-client-utils.test.ts` was added to cover:
* Successful conversion of standard Markdown headers, bold, italics, and lists to HTML.
* Stripping of dangerous attributes like `onerror`.
* Stripping of `javascript:` protocol links to prevent XSS.
* Complete removal of `<script>` tags, preventing injection.
* Proper handling of `<img>` tags (which are explicitly allowed).
* Conversion of newlines to `<br>` tags (`breaks: true`).
* Handling of empty strings.

✨ **Result:** 100% of the `textToHtml` utility logic is now backed by unit tests, establishing a safety net for its sanitization logic and configuration options.
